Se describirá acontinuación los pasos para clonar un proyecto Laravel de un repositorio como Github
Se deben tener los servicios de Apache y MySQL activados

1. Descargar el repositorio en la carpeta IngSoftware

	$ git clone https://github.com/Arbert407/Ingenieria-de-Software.git/ IngSoftware

2. Moverse hasta la carpeta del proyecto (Proyecto_COCESNA)

	$ cd IngSoftware/Proyecto_COCESNA

3. Instalar las dependencias de Composer
	
	$ composer install

4. Instalar dependencias NPM

	$ npm install

5. Renombrar el archivo .env.example a .env y borrar la cache con el siguiente comando:

	$ php artisan config:clear

6. Modificar las variables en el archivo .env con la siguiente información:
	
	APP_NAME=Laravel
	APP_ENV=local
	APP_KEY=base64:p7nks91KjyjeC+nJRC2LIdFQBqRqT6/RC0+D0PLmmoQ=
	APP_DEBUG=true
	APP_URL=http://localhost

7. Modificar las variables en el archivo .env con la siguiente información:

	DB_CONNECTION=mysql
	DB_HOST=localhost
	DB_PORT=3306
	DB_DATABASE=db_is
	DB_USERNAME=NOMBRE-DE-USUARIO
	DB_PASSWORD=CONTRASEÑA-DE-LA-BASE-DE-DATOS

	Para usar el correo modificamos las siguientes variables:
	
	MAIL_DRIVER=smtp
	MAIL_HOST=smtp.googlemail.com
	MAIL_PORT=465
	MAIL_USERNAME=rrhh.notifier@gmail.com
	MAIL_PASSWORD=dBgv%434c223!@sa
	MAIL_ENCRYPTION=ssl

	Por lo general el nombre de usuario es root y si no se tiene contrasena se deja en blanco ('')

8.	Crear la base de datos, para ello tenemos 2 formas: 
	a) Buscar db_is.sql en la carpeta "Proyecto_COCESNA" y ejecutarlo en PHPMyAdmin  
    b) Desde MySQL Workbench abrir archivo db_is.mwb y presionar Ctrl+G para crear la base de datos 

9. Llenar la base con datos de prueba y clonar la base para realizar el testeo

	$ php artisan db:seed

10. Iniciar el servidor de prueba

	$ php artisan serve
